# HAX-CODE — تسليم العميل / اختبارات القبول

هذا الملف يربط **اختبارات Playwright الآلية** بـ **قائمة تحقق يدوية** قبل التسليم.

## تشغيل الاختبارات الآلية

```bash
cd hax-code-website
npm install
npx playwright install chromium   # مرة واحدة على الجهاز
npm run test:e2e
```

- تقرير HTML بعد الفشل: `npm run test:e2e:report`
- مع متصفح مرئي: `npm run test:e2e:headed`
- في CI: عيّن `CI=1` (إعادة المحاولة + خادم جديد)

## قائمة تحقق يدوية (لا تغني عن الآلي)

- [ ] **موبايل / تابلت:** القائمة، الهيرو، والأزرار قابلة للاستخدام
- [ ] **محتوى:** استبدال placeholders البريد / الهاتف / العنوان في الفوتر
- [ ] **روابط CTA:** ربط «احجز مكالمة» و«اطلب عرضاً» بروابط حقيقية (تقويم، واتساب، بريد)
- [ ] **أداء:** Lighthouse سريع على 3G (اختياري)
- [ ] **متصفحات:** Safari + Chrome على الأقل (الآلي حالياً Chromium فقط)

## ملاحظات تقنية

- عناصر الاختبار: `data-testid` على الهيدر، تبديل اللغة، المظهر، وأقسام الهيرو / العملية / الأسئلة.
- الخادم المحلي يُشغَّل تلقائياً عبر `playwright.config.ts` على `127.0.0.1:5173` ما لم يُضبط `PORT`.
